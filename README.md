# E13 OCR Bot

Telegram-бот для распознавания текста с изображений с помощью **Qwen Portal Vision API**.

## Возможности

- Распознавание текста с фотографий (сжатых Telegram)
- Распознавание текста с документов-изображений (оригинальное качество)
- Поддержка форматов: JPEG, PNG, WebP, GIF
- Результат в формате Markdown с сохранением структуры
- Индикатор обработки и обработка ошибок
- Автоматическое чтение актуального OAuth-токена из файла (без перезапуска)

## Требования

- Docker и Docker Compose
- Telegram Bot Token (от [@BotFather](https://t.me/BotFather))
- Qwen CLI (для авторизации и получения OAuth-токена)

## Использование

1. Найдите бота в Telegram и нажмите **Start** (или отправьте `/start`)
2. Отправьте фотографию или изображение как документ
3. Бот покажет `⏳ Обрабатываю...` и через несколько секунд вернёт распознанный текст

### Пример

```
Вы: [отправляете фото чека]

Бот: ⏳ Обрабатываю...

Бот: 📝 Текст:

## Чек
- Товар 1 — 150 ₽
- Товар 2 — 230 ₽
- **Итого: 380 ₽**
```

## Установка на сервере

### 1. Установка Docker и Docker Compose

```bash
# Установка Docker (если не установлен)
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# Установка Docker Compose (если не установлен)
sudo apt-get update
sudo apt-get install docker-compose-plugin
```

### 2. Получение Qwen Auth Token

Перед запуском бота необходимо установить Qwen CLI и пройти авторизацию.

#### 2.1 Установка Qwen CLI

```bash
curl -fsSL https://qwen-code-assets.oss-cn-hangzhou.aliyuncs.com/installation/install-qwen.sh | bash
```

Переподключитесь к серверу (выйдите и войдите снова), затем проверьте установку:

```bash
qwen --version
```

#### 2.2 Авторизация

```bash
qwen
```

Запустится Qwen CLI. Выберите **Qwen OAuth**, скопируйте ссылку авторизации, вставьте в браузер и нажмите «Авторизоваться».

После успешной авторизации токен автоматически сохраняется в файл:

```
~/.qwen/oauth_creds.json
```

Проверить содержимое:

```bash
cat ~/.qwen/oauth_creds.json
```

> **Важно:** Этот файл монтируется в контейнер через `docker-compose.yml`. Бот читает `access_token` из него при каждом запросе к API, поэтому при обновлении токена перезапуск контейнера не требуется.

### 3. Автоматическое обновление токена (cron)

Токен Qwen действителен **6 часов**. В репозитории есть скрипт `refresh_qwen_token.py`, который проверяет срок действия и при необходимости обновляет токен через API.

```bash
# Скопируйте скрипт (если ещё не в корне проекта)
# Убедитесь, что он исполняемый
chmod +x refresh_qwen_token.py

# Проверьте работу вручную
python3 refresh_qwen_token.py
```

Добавьте задачу в cron (обновление каждый час):

```bash
crontab -e
```

Добавьте строку:

```
0 * * * * /usr/bin/python3 /root/e13ocrbot-1/refresh_qwen_token.py >> /var/log/qwen_refresh.log 2>&1
```

> **Примечание:** Скрипт обновляет токен, только если до его истечения осталось менее 2 часов. Если токен ещё действителен — ничего не делает. Перезапуск бота не требуется — он читает файл при каждом запросе.

### 4. Клонирование и настройка проекта

```bash
# Клонирование репозитория
git clone <repo-url>
cd e13ocrbot

# Создание файла .env
cp .env.example .env
nano .env
```

Содержимое `.env`:

```
TELEGRAM_TOKEN=ваш_токен_от_BotFather
```

### 5. Запуск контейнера

С помощью docker-compose (рекомендуется):
```bash
docker-compose up -d
```

> При запуске директория `~/.qwen/` с хоста монтируется в контейнер по пути `/app/qwen_creds/` (read-only). Бот читает токен из `/app/qwen_creds/oauth_creds.json`. Если файл не найден — бот не запустится.

### 6. Просмотр логов
```bash
# С docker-compose
docker-compose logs -f bot

# Или напрямую
docker logs -f e13ocrbot
```

### 7. Управление контейнером

**Остановка:**
```bash
docker-compose down
# или
docker stop e13ocrbot && docker rm e13ocrbot
```

**Перезапуск:**
```bash
docker-compose restart
```

**Проверка статуса:**
```bash
docker ps | grep e13ocrbot
```

### 8. Пересборка образа

После изменений в коде или зависимостях:
```bash
docker-compose build
docker-compose up -d
```

## Монтируемые файлы

| Путь на хосте    | Путь в контейнере      | Описание                                    |
| ---------------- | ---------------------- | ------------------------------------------- |
| `~/.qwen/`       | `/app/qwen_creds/`     | Директория с OAuth-токеном Qwen (read-only) |

Бот читает `access_token` из файла `oauth_creds.json` внутри этой директории при каждом запросе к API.

> **Важно:** Монтируется именно **директория**, а не файл. Это решает проблему кэширования: при монтировании отдельного файла Docker привязывается к его inode, и при перезаписи файла (новый inode) контейнер не видит изменений.

## Обработка ошибок

Бот обрабатывает следующие ситуации:

- ⏱ **Таймаут API** — сообщение пользователю о повторной попытке
- 🔑 **Ошибка авторизации (401)** — предупреждение о неверном токене
- 🚦 **Лимит запросов (429)** — просьба подождать
- 🔧 **Ошибка сервера (5xx)** — информация о временной недоступности
- ⚠️ **Неверный формат файла** — подсказка отправить изображение

Все ошибки логируются в консоль с подробностями.
